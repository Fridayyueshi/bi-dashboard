### M1 — 核心KPI 模块

**职责：** 展示店铺近7日核心经营指标 + 上周同比

**数据源：** `sycm_sp_all`

**API：** `GET /api/kpi`

**返回字段：**
```json
{
  "payment_amount": 656265.55,
  "payment_items": 910,
  "buyers": 207,
  "refund_amount": 348711.21,
  "amount_trend": "30.3%",
  "items_trend": "181.7%"
}
```

**前端组件：** 4个 KPI 卡片（支付金额、支付件数、买家数、退款金额），含环比箭头

**完善方向：**
- [ ] 增加"日均"指标（总额/7）
- [ ] 增加"转化率"指标（支付买家数/访客数）
- [ ] 增加"客单价"指标（支付金额/支付买家数）
- [ ] 增加"退款率"指标（退款金额/支付金额）
- [ ] 环比周期可配置（周环比/月环比）

---

### M2 — 日趋势图模块

**职责：** 近7日每日销售额和件数趋势

**数据源：** `sycm_sp_all`

**API：** `GET /api/trend`

**返回字段：**
```json
{
  "dates": ["2026-05-24", ...],
  "amounts": [67659.61, ...],
  "items": [40, ...]
}
```

**前端组件：** 柱状图（纯 CSS，无第三方库），双Y轴可选

**完善方向：**
- [ ] 增加"件数"柱状图（双柱对比）
- [ ] 增加"买家数"折线叠加
- [ ] 增加"退款金额"折线叠加
- [ ] 支持切换时间范围（7日/30日/自定义）
- [ ] 鼠标悬停显示具体数值
- [ ] 点击柱子跳转到当日明细

### M3 — 商品TOP5模块

**职责：** 近7日支付金额TOP5商品排行

**数据源：** `sycm_sp_all`

**API：** `GET /api/product_top`

**返回字段：**
```json

  {"id": 820012962604, "name": "南巢黑胡桃木...", "amount": 194874.92, "items": 53, "share": 55.8}
]
```

**前端组件：** 表格 + 排名徽章 + 占比进度条

**完善方向：**
- [ ] 增加"环比变化"列（对比上周排名变化）
- [ ] 增加"新上架"标签
- [ ] 增加"售罄"标签
- [ ] 点击商品跳转到商品详情页（外部链接）
- [ ] 支持切换"件数TOP"和"金额TOP"
- [ ] TOP N 可配置（5/10/20）

**前端组件：** 表格 + 点击量占比进度条

**完善方向：**
- [ ] 增加"展现量"和"花费"占比
- [ ] 计算并展示"点击率"（clicks/impressions）

- [ ] 增加"投产比"列（需关联成交金额）
- [ ] 支持按"花费"排序

- [ ] 计划层级 → 单元层级下钻

## 三、模块依赖关系

```
                    ┌─────────────┐
                    │  数据库 PG   │
                    └──────┬──────┘
                           │
         ┌─────────────────┼─────────────────┐
         │                 │                 │
    ┌────┴────┐       ┌────┴────┐      ┌────┴────┐
    │ sycm_*  │       │ wj_*    │      │competitor│
    │ (生意参谋)│      │ (无界)   │      │  (竞品)  │
    └────┬────┘       └────┬────┘      └────┬────┘
         │                 │                 │
    ┌────┴────┐       ┌────┴────┐      ┌────┴────┐
    │ M1 KPI  │       │ M4 流量 │      │ M7 竞品 │
    │ M2 趋势 │       │ M5 关键词│      │ M8 SKU  │
    │ M3 TOP5 │       │ M6 报表 │      │         │
    └────┬────┘       └────┬────┘      └────┬────┘
         │                 │                 │
         └─────────────────┼─────────────────┘
                           │
                    ┌──────┴──────┐
                    │ 前端 SPA    │
                    │ (统一渲染)  │
                    └─────────────┘
```

**关键依赖：**
- M7/M8 依赖竞品监控 cron 任务（M11）先运行产生数据
- M9 导出模块依赖所有数据模块
- M10 多店铺是架构级变更，影响所有模块

## 四、开发优先级

## 五、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 原生 HTML/CSS/JS | 单文件 SPA，无构建步骤，无第三方框架 |
| 后端 | Flask 3.0 + Flask-CORS | 轻量级 REST API |
| 数据库 | PostgreSQL 15 | Windows 主机，WSL 访问 |
| 定时任务 | Hermes cron | 每日自动抓取竞品数据 |
| 部署 | 直接运行 | `python bi_dashboard_server.py` |
